## 后端逻辑层

### 音高检测管线

```
麦克风 PCM → 预处理 → 基频检测 → 音符映射 → 平滑输出
```

**基频检测算法选型**（这是核心）：

| 算法         | 延迟 | 低频准确度 | 适用乐器                 |
| ------------ | ---- | ---------- | ------------------------ |
| YIN          | 低   | ★★★★☆      | 弦乐、人声（推荐小提琴） |
| MPM (McLeod) | 低   | ★★★★★      | 弦乐最优                 |
| HPS          | 极低 | ★★☆☆☆      | 吉他泛音丰富时才准       |
| CREPE (ML)   | 高   | ★★★★★      | 嘈杂环境，但实时性差     |

小提琴场景推荐 **MPM**，它是 YIN 的改进版，对小提琴 G 弦（196Hz）这类低频更准确。

**平滑策略**很关键——原始检测值抖动严重，需要：

```cpp
// 中位数滤波（去离群值）+ 指数平滑（去抖动）
float median_filter(float* history, int n);  // 取最近5帧中位数
float ema = alpha * raw + (1 - alpha) * prev_ema;  // alpha ≈ 0.15
```

**置信度过滤**：低于阈值（通常 0.85）的检测结果直接丢弃，不更新 UI。

### UI 状态机详解

专业软件的精髓在于**状态机驱动 UI**，而不是直接把原始检测值塞给显示层：

**四个状态的视觉映射：**

| 状态        | 音符显示         | 指针     | 颜色   |
| ----------- | ---------------- | -------- | ------ |
| `idle`      | `--`             | 隐藏     | 灰色   |
| `detecting` | 闪烁当前候选音符 | 抖动     | 蓝/青  |
| `locked`    | 稳定显示         | 平滑移动 | 紫     |
| `in-tune`   | 稳定显示         | 居中     | 绿色 ✓ |

**locked → in-tune 的判断逻辑**（常被忽略的细节）：

```dart
// 不是瞬间判断，而是"持续在准的区间内"
bool isInTune = recentCents.every((c) => c.abs() < 2.0)
             && recentCents.length >= 5;  // 连续5帧 ≈ 125ms
```

这避免了指针"掠过零点"时误触绿色反馈。

### 偏差显示的三种流派

**流派 A：指针式（Peterson、Cleartune）**

模拟表盘，±50¢ 范围，中间绿区约 ±2¢。心理上最接近物理调音器，专业用户偏好。关键实现：指针动画用 `lerp` 而非直接跳变，响应时间约 80ms。

**流派 B：进度条 / 弧形（GuitarTuna）**

横向色条，左红→中绿→右红。更直觉，适合初学者。实现简单，但对精确偏差（±5¢ 以内）分辨率不够。

**流派 C：频闪节拍法（Strobosoft）**

将偏差转换成条纹移动速度，静止=准音。精度可达 ±0.1¢，但对小提琴初学者太陌生。

**你的场景建议**：指针式为主，初学模式可切换为进度条，是个合理的分层设计。

### 音符映射逻辑（Dart 层做这个）

```dart
// 从频率到 {音符名, 八度, 偏差(cents)}
TunerResult frequencyToNote(double freq) {
  if (freq <= 0) return TunerResult.empty;

  // A4 = 440Hz 为基准，可被用户调整（如 A4 = 432Hz）
  final semitones = 12 * log(freq / a4Reference) / log(2);
  final nearestSemitone = semitones.round();
  final cents = (semitones - nearestSemitone) * 100;

  // MIDI 音符号 → 音符名 + 八度
  final midiNote = 69 + nearestSemitone;  // A4 = MIDI 69
  final noteName = noteNames[midiNote % 12];
  final octave = (midiNote / 12).floor() - 1;

  return TunerResult(note: noteName, octave: octave, cents: cents);
}
```

这里有个**重要的用户体验细节**：`a4Reference` 要让用户可以调整（432/440/441/442 Hz），演奏者和乐团一起练习时经常需要，小提琴手尤其如此。

### 小提琴场景的特殊考量

小提琴有几个调音器设计中要特别处理的地方：

1. **弓弦噪声**：起弓瞬间有宽频噪声，置信度阈值要比吉他高（≥0.88 vs 0.85）
2. **泛音丰富**：G弦基频 196Hz，但第一泛音 392Hz 能量常常更强，MPM 比 YIN 在这里更不容易"跳八度"
3. **揉弦（Vibrato）识别**：专业软件会把周期性的 ±20¢ 抖动识别为 vibrato，显示中心频率而非瞬时值。这是进阶功能，可以二期做
4. **四弦标记**（G D A E）：在 UI 上常驻显示当前检测到的音最接近哪根弦，帮助初学者建立空间感
