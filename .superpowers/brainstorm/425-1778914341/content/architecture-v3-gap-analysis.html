对照吴恩达 Agentic AI — 架构差距分析

逐项对比当前设计 vs 课程最佳实践，找出迭代方向

差距 1：缺少 Evals + Trace 基础设施（⚠ 最关键的遗漏）
"决定团队能否高效构建 Agent 的最大因素，是能否执行严格的 Evals 和错误分析流程。" — 吴恩达
当前状态 应该增加
❌ 无评测体系 ✅ Trace Logger — 记录每个 Agent 调用的输入/输出/耗时/成功失败
✅ Eval Engine — 客观评测（音准偏差%）、主观评测（LLM 评分）、二元判断
✅ Trace Viewer UI — 在 App 内可视化 Agent 执行轨迹
差距 2：规划引擎只支持一种策略
当前状态 应该增加
⚠ 只有 Plan-then-Execute（预先生成计划 → 执行） ✅ 增加 ReAct 模式（Reason → Act → Observe → 循环）
适用场景："帮我改善音色" — 无法预定义步骤，需一边练一边调整
差距 3：反思是单模型，应该是双模型
当前状态 应该改进
⚠ 同一个 LLM 即生成又审查 ✅ Generator Agent（轻量模型，如 Claude Haiku / GPT-3.5）生成练习计划
✅ Critic Agent（强模型，如 Claude Opus / GPT-4）审查计划质量
课程数据：GPT-3.5 + 反思循环 → 74%，超过 GPT-4 直接输出 67%
差距 4：编排拓扑单一
当前状态 应该增加
⚠ 只有层级式（Orchestrator → Agents） ✅ 对等式（P2P） — Technique Analyst ↔ Repertoire Advisor 互相协商
✅ Agent-as-Tool — 一个 Agent 把另一个 Agent 当工具调用
✅ 图式状态机 — 借鉴 LangGraph，用状态图替代树形编排
差距 5：缺少 MCP 协议支持
当前状态 应该改进
⚠ 每个 Tool 通过自定义 Dart 接口暴露 ✅ 插件 额外暴露 MCP Server 接口，LLM 通过标准协议发现和调用工具
好处：未来可接入外部 MCP 工具（如联网搜谱、云端伴奏库）
差距 6：缺少 Human-in-the-loop 节点
当前状态 应该增加
❌ Agent 决策无人工确认环节 ✅ 关键节点暂停等待用户确认（如："本周练习计划已生成，确认开始？"）
✅ 可配置自主性级别（完全自动 / 半自动 / 纯手动）

6 项改进总结

#改进项来源优先级
1Trace Logger + Eval Engine + Trace Viewer §2 评测与错误分析 🔴 必须
2双规划策略：Plan-then-Execute + ReAct §3 规划模式 🔴 必须
3双模型反思：Generator(轻量) + Critic(强模型) §3 反思模式 🔴 必须
4状态图编排：图节点+边，替代纯树形 §5 LangGraph 🟡 推荐
5MCP 标准接口 + 外部工具集成 §3 MCP 🟡 推荐
6Human-in-the-loop + 自主性级别配置 §6 人机协同 🟡 推荐

最关键的三项（1+2+3）直接来自吴恩达课程的强制性最佳实践，其余三项是可选的增强。打开 http://localhost:57439 查看完整对比。
